# 智能回复技术调研 — 文字 & 语音

> **调研日期：** 2026-07-22
> **调研目的：** 梳理「智能文字回复」和「智能语音回复」的技术实现方案，回答"是否需要训练模型""如何实现""当前设置是否合理"三个核心问题。
> **结论先行：** 文字回复不需要训练模型（LLM+RAG 即可），语音回复基础能力不需要训练（TTS API），但音色克隆需要。当前 Demo 的互动设置基本合理，缺少"知识库管理"模块。

---

## 一、智能文字回复

### 1.1 要不要训练模型？

**不需要。**

文字回复的核心是**大语言模型（LLM）+ 检索增强生成（RAG）**，不是从头训练一个模型。

```
观众弹幕 → 意图识别 → 知识库检索 → LLM 生成回复 → 输出文字
```

每一步都基于现成能力：
- **意图识别**：用轻量 NLP 模型分类（提问/闲聊/下单/投诉），不需要训练，规则+小模型即可
- **知识库检索**：向量数据库（Milvus）+ embedding 模型（如 text2vec），不需要训练
- **LLM 生成**：调用 DeepSeek / 通义千问 / GPT API，不需要训练

### 1.2 那"训练"到底指什么？

如果有人说"需要训练"，通常指以下三件事之一，但它们都**不是传统意义的模型训练**：

| 做的事 | 实际是什么 | 是否需要 GPU |
|------|------|:---:|
| "训练 AI 了解我的商品" | 把商品详情、FAQ、脚本写入知识库（RAG） | ❌ |
| "训练 AI 用我的语气说话" | 写 System Prompt + Few-shot 示例 | ❌ |
| "训练一个专属模型" | Fine-tuning LLM（极少需要） | ✅（仅大厂做） |

**结论**：99% 的场景只需要做好知识库 + Prompt 工程，不需要训练模型。

#### 关键实现细节

**1. 意图分类（决定回不回复、怎么回）**

| 意图 | 示例 | 策略 |
|------|------|------|
| 商品提问 | "这个有XL吗？" | 检索知识库 → LLM 生成 |
| 价格咨询 | "多少钱？" | 直接查价格 → 模板化回复 |
| 下单意向 | "已下单！" | 感谢 + 物流信息 |
| 闲聊 | "主播好漂亮" | 简单寒暄 / 不回复 |
| 投诉 | "上次买的坏了" | 转人工 / 售后话术 |

→ 对应 Demo 中的「回复决策：仅回复提问 / 回复所有」

**2. 知识库检索（RAG）**

- 将商品详情、规格参数、价格、FAQ 向量化存入 Milvus
- 弹幕到达时检索最相关的知识片段
- 把检索结果作为 Context 注入 LLM Prompt

→ 对应 Demo 中的「知识库来源」多选

**3. LLM 生成**

System Prompt 示例：
```
你是直播间AI主播助手。根据以下商品信息回答观众问题。
- 回复简短（50字以内），口语化，带"亲"等亲切称呼
- 不知道的就说不确定，不要编造
- 遇到投诉请引导联系客服

商品信息：{检索结果}
历史对话：{最近5轮}

观众：这个有XL码吗？
主播：
```

→ 对应 Demo 中的「回复风格」

## 二、智能语音回复

### 2.1 要不要训练模型？

**基础 TTS 不需要，音色克隆需要。**

```
文字回复 → TTS 引擎 → 音频流 → 混入推流
```

- **基础 TTS**：阿里云 CosyVoice / 讯飞 / Azure TTS，直接调用 API，输入文字输出音频，**零训练**
- **音色克隆**：需要提供目标说话人的音频样本（通常 10 分钟以上），训练/微调声学模型。这是本项目 Phase 3 的事

### 2.2 两种语音模式

| 模式 | 原理 | 效果 | 实现难度 |
|------|------|------|:---:|
| **原音频驱动** | 从预录视频中提取真人语音片段，拼接成回复 | 音色自然，但衔接生硬，词汇受限 | 中 |
| **TTS 驱动** | 文字 → TTS API 合成语音 | 任意文字都能读，但音色可能和真人不同 | 低 |

→ 对应 Demo 中的「回复语音模式：原音频驱动 / TTS 驱动」

### 2.3 实现方案

```
文字回复输出
     │
     ▼
┌────────────┐    ┌──────────────┐    ┌──────────┐
│ TTS 引擎    │ →  │ 音频后处理    │ →  │ 混入推流  │
│            │    │ (音量/语速)   │    │ (FFmpeg) │
│ CosyVoice2 │    │              │    │          │
│ 阿里云TTS  │    │ DSP 降噪     │    │ 替换BGM  │
└────────────┘    └──────────────┘    └──────────┘
```

→ 对应 Demo 中的「语速/音量/语调调节」「TTS 读法修正」

## 三、当前 Demo 设置评估

### 3.1 已有的设置（互动设置页面）

| 设置项 | 合理性 | 说明 |
|--------|:---:|------|
| 回复语音模式（原声/TTS） | ✅ 合理 | 这是核心技术选择，Phase 2 应该都有 |
| 语速/音量/语调 | ✅ 合理 | TTS 的标准参数 |
| 回复决策（仅提问/全部） | ✅ 合理 | 对应意图分类的策略选择 |
| TTS 读法修正 | ✅ 合理 | 行业通用需求（多音字、英文、数字读法） |
| 文字回复开关 | ✅ 合理 | 总开关 |
| 知识库来源（商品/手动/脚本/全购物车） | ✅ 合理 | 对应 RAG 的数据源选择 |
| 回复风格（亲切/专业/活泼） | ✅ 合理 | 对应 System Prompt 风格 |
| 衔接模式（承上启下/独立） | ✅ 合理 | 对应是否带对话历史 |
| 单品互动话术模板 | ✅ 合理 | 对应 FAQ 级别的精确匹配 |
| 定时弹幕/主动互动 | ✅ 合理 | 场控自动化 |

### 3.2 缺少的

| 缺少项 | 重要性 | 说明 |
|--------|:---:|------|
| **知识库管理页面** | 🔴 高 | 目前只有"来源勾选"，缺少知识库内容的查看、编辑、测试入口 |
| **回复预览/测试** | 🟡 中 | 输入一条模拟弹幕，实时看到 AI 会回复什么 |
| **敏感词过滤** | 🟡 中 | 已在 nlp-svc 中规划，但 Demo 没体现 |
| **回复延迟统计** | 🟢 低 | 运营总览有 AI 回复率，但没有延迟数据 |

### 3.3 建议新增：知识库管理

在 Demo 的互动设置旁边或内容管理下，增加一个「知识库管理」入口：

```
📚 知识库管理
  ├── 商品知识（自动从商品知识库同步）
  ├── 自定义 FAQ（手动添加问答对）
  ├── 话术模板（批量导入/导出）
  └── 测试对话（输入弹幕 → 预览 AI 回复）
```

## 四、总结

| 问题 | 答案 |
|------|------|
| 文字回复需要训练模型吗？ | **不需要。** LLM API + RAG 知识库即可，核心工作是建知识库和写 Prompt |
| 语音回复需要训练模型吗？ | **基础不需要。** TTS API 零训练。音色克隆才需要（Phase 3） |
| 当前设置合理吗？ | **基本合理，** 覆盖了 LLM + TTS 的核心参数。缺少知识库管理页面 |
| 怎么实现？ | 文字：弹幕→意图→RAG检索→LLM生成。语音：文字→TTS→混流 |
