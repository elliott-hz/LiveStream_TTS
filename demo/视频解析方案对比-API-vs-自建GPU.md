# 视频 AI 解析方案：两套技术路线 × 三种成本模型

> **输入**：商户真人商品讲解视频，平均 1h/个，1080p
> **输出**：切片视频 + 脚本文字 + TTS 音色数据 + SKU 关联
> **规模**：3000 个视频/周，单视频 ≤ 5h 完成

---

## 一、技术路线 A：全 API（无 GPU）

### 执行过程

```
商户上传视频 → OSS 对象存储
                  │
                  ▼
┌─────────────────────────────────────────┐
│  Step 1: ASR 语音识别                    │
│  阿里云实时语音识别 API                   │
│  POST audio → 返回全文稿 + 时间戳         │
│  耗时: ~3min    费用: ¥3.5/视频          │
├─────────────────────────────────────────┤
│  Step 2: 场景/商品边界检测                │
│  阿里云智能媒体 API (SubmitMediaAIJob)    │
│  分析镜头切换、画面变化、OCR文字变化        │
│  耗时: ~15min   费用: ¥7.5/视频          │
│  ⚠️ 非电商专用，复杂场景准确率 ~70%        │
├─────────────────────────────────────────┤
│  Step 3: 脚本提取 + 清洗                  │
│  DeepSeek API / 通义千问 API             │
│  ASR 全文稿 → 按商品边界分段 → 去口语词    │
│  耗时: ~30s     费用: ¥0.001/视频        │
├─────────────────────────────────────────┤
│  Step 4: 视频切割                         │
│  普通 CPU 服务器 FFmpeg stream copy       │
│  ffmpeg -ss start -to end -c copy        │
│  耗时: ~2min    费用: 忽略（电费）         │
├─────────────────────────────────────────┤
│  Step 5: TTS 音色克隆（可选）              │
│  阿里云 CosyVoice API                    │
│  Few-shot: 参考音频 → 即时复刻音色         │
│  耗时: ~5min    费用: ¥1/视频             │
│  精调:   完整音频 → 微调声学模型           │
│  耗时: ~2h      费用: ¥500/视频           │
└─────────────────────────────────────────┘
                  │
                  ▼
        输出 → OSS → 商户端拉取
```

### 关键技术风险

**场景检测是短板**。阿里云智能媒体 API 能检测镜头切换，但不是为「商品讲解场景」训练的。实际效果：

- 主播有明显停顿 + 换品动作 → 准确率 ~85%
- 主播手持商品自然切换 → 准确率 ~60%
- 需要商户手动微调的概率 → 约 30%

**补救方案**：不做 AI 场景检测，用纯 ASR 分段。检测到"接下来给大家看..." 这类切换关键词就切。准确率更低但零额外成本。

## 二、技术路线 B：GPU 推理（自建 or 云租用，执行过程相同）

### 执行过程

```
商户上传视频 → OSS 对象存储 / 本地 NAS
                  │
                  ▼
┌─────────────────────────────────────────┐
│  Step 1: ASR 语音识别                    │
│  Whisper large-v3（GPU 本地推理）         │
│  模型: openai/whisper-large-v3           │
│  GPU 耗时: ~6min / 1h视频 (T4)           │
│  1×T4 每小时处理: ~10 个视频              │
├─────────────────────────────────────────┤
│  Step 2: 场景/商品边界检测                │
│  CLIP ViT-L/14（GPU 本地推理）            │
│  每5秒抽1帧 → CLIP 计算帧间相似度          │
│  相似度骤降 = 商品切换边界                 │
│  GPU 耗时: ~10min / 1h视频 (T4)          │
│  1×T4 每小时处理: ~6 个视频               │
│  ✅ 准确率 ~90%，可持续微调优化            │
├─────────────────────────────────────────┤
│  Step 3: 脚本提取 + 清洗                  │
│  DeepSeek API（同方案 A）                 │
│  耗时: ~30s     费用: ¥0.001             │
├─────────────────────────────────────────┤
│  Step 4: 视频切割                         │
│  CPU 服务器 FFmpeg stream copy            │
│  ffmpeg -ss start -to end -c copy        │
│  耗时: ~2min                             │
├─────────────────────────────────────────┤
│  Step 5: TTS 音色克隆                    │
│  CosyVoice2（GPU 本地推理）               │
│  Few-shot: 3-10s 参考音频 → 即时复刻      │
│  GPU 耗时: ~5min                         │
│  精调: GPT-SoVITS（GPU 训练）             │
│  GPU 耗时: ~90min / 视频                 │
└─────────────────────────────────────────┘
                  │
                  ▼
        输出 → 存储 → 商户端拉取
```

### 任务调度架构

```
┌──────────────────────────────────────────────┐
│              任务队列 (Celery / Redis)          │
│                                              │
│  视频1 ──→ [ASR] ──→ [CLIP] ──→ [LLM] ──→ [FFmpeg] ──→ [TTS]
│  视频2 ──→ [ASR] ──→ [CLIP] ──→ [LLM] ──→ [FFmpeg] ──→ [TTS]
│  ...                                         │
│  视频N ──→ [ASR] ──→ [CLIP] ──→ [LLM] ──→ [FFmpeg] ──→ [TTS]
│                                              │
│  并行度: GPU 卡数 × 每卡并发 2-3 个任务        │
│  单视频 GPU 总时间: ~21min (基础) / ~111min (精调) │
└──────────────────────────────────────────────┘
```

### GPU 规模计算

**基础解析（含 TTS Few-shot，21min/视频）**：
- 3000 × 21min = 63,000 min = **1,050 GPU-hours**
- 168h 内完成：1,050 ÷ 168 = 6.3 → **8 张 T4**（含余量）

**含 10% TTS 精调（300 个精调，111min/视频）**：
- 2700 × 21min + 300 × 111min = 56,700 + 33,300 = **1,500 GPU-hours**
- 168h 内完成：1,500 ÷ 168 = 8.9 → **12 张 T4**

**全部精调（3000 个都精调，111min/视频）**：
- 3000 × 111min = **5,550 GPU-hours**
- 168h 内完成：5,550 ÷ 168 = 33 → **40 张 T4**

| 场景 | GPU 数 | 服务器 |
|------|:---:|------|
| 基础 | 8× T4 | 2 台 |
| 10% 精调 | 12× T4 | 3 台 |
| 100% 精调 | 40× T4 | 10 台 |

## 四、存储成本核算

### 单个视频的存储构成

以 1 个 2h / 1080p / H.264 视频为例：

| 文件 | 大小 | 说明 |
|------|:---:|------|
| 原始视频（上传） | ~3 GB | H.264 1080p, 3.5 Mbps 典型码率 |
| 切片视频（输出） | ~3 GB | Stream copy，总大小 = 原始 |
| 缩略图（5张/视频） | ~2.5 MB | 每张 500KB |
| 脚本文稿 | ~10 KB | 纯文本 |
| **单视频合计** | **~6 GB** | 含原始+切片 |

### TTS 音色模型（按商户，不按视频）

| 类型 | 大小 | 说明 |
|------|:---:|------|
| CosyVoice2 Few-shot | ~50 MB | 参考音频 + 音色向量，一个商户一份 |
| GPT-SoVITS 精调 | ~200 MB | 完整声学模型，一个商户一份 |

> TTS 模型按商户存储，同一个商户的所有视频共用一份音色数据。

### 商户级存储估算

| 商户规模 | 视频数 | 存储量 |
|------|:---:|------|
| 小微商户 | 10 个 | ~60 GB |
| 中型商户 | 50 个 | ~300 GB |
| 大型商户 | 100 个 | ~600 GB |
| 超大型商户 | 200 个 | ~1.2 TB |

### 3000 视频总存储

| 项目 | 计算 | 容量 |
|------|------|:---:|
| 3000 个视频（原始+切片） | 3000 × 6GB | 18 TB |
| 缩略图 + 脚本 | 3000 × 2.5MB | ~7.5 GB |
| TTS 模型（假设 100 商户） | 100 × 200MB | 20 GB |
| **总计** | | **≈ 18 TB** |

加上操作系统、日志、临时转码空间、RAID 冗余，实际需要 **25-30 TB** 可用容量。

### 存储分层策略

```
┌─────────────────────────────────────┐
│  热存储（SSD/NVMe）                   │
│  正在处理的视频 + 活跃商户近7天视频     │
│  ~2TB · 贵但快                       │
├─────────────────────────────────────┤
│  温存储（HDD NAS）                    │
│  近30天的原始视频 + 全部切片视频        │
│  ~18TB · 主要存储层                   │
├─────────────────────────────────────┤
│  冷存储（OSS 低频/归档）               │
│  超过30天的原始视频（极少访问）          │
│  按需扩容 · 极便宜                     │
└─────────────────────────────────────┘
```

### 存储成本对比

| 存储方式 | 单价 | 18TB 月费 | 30TB 月费 |
|------|------|:---:|:---:|
| **自建 NAS**（HDD RAID5） | — | ¥0（已含硬件） | ¥0（已含硬件） |
| **OSS 标准存储** | ¥0.12/GB | ¥2,160 | ¥3,600 |
| **OSS 低频存储** | ¥0.08/GB | ¥1,440 | ¥2,400 |
| **OSS 归档存储** | ¥0.03/GB | ¥540 | ¥900 |

> 自建 NAS 的 20TB 已在硬件成本中包含（¥15,000），后续只耗电费和硬盘替换。

```

┌──────────────────────────────────────────┐
│  云方案：OSS 标准 + 低频混合                │
```

## 五、建议

```
PoC 验证（当前）
  → 路线 3 云 GPU 竞价实例，跑 100 个视频验证效果
  → 月费 ~¥2,000

量起来后（稳定 500+ 视频/周）
  → 切换到路线 2 自建，8-12 张 T4
  → 第一年回本（vs API），之后纯赚
  → 场景检测模型可持续微调，越用越准

永远不推荐路线 1 长期跑
  → 三年成本 560 万 vs 自建 95 万，6 倍差距
  → 场景检测准确率还更低
```

> **一句话**：API 起步做验证 → 云 GPU 跑 PoC → 自建 GPU 跑规模化。
