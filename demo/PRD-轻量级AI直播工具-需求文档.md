## 1. 产品概述

### 1.1 产品定义

一款**云端 SaaS 直播工具**（仅云端，不提供本地私有化部署）。商家在网页端完成商品管理、视频素材 AI 解析、直播视频编排、直播间搭建后，一键推流至电商平台，系统自动循环播放编排好的商品讲解视频，同时 AI 实时回复观众弹幕。

### 1.2 核心价值

| 痛点 | 解决方案 |
|------|---------|
| 真人主播成本高、时长受限 | 预录商品视频 + 播单循环，7×24 小时无人值守 |
| 弹幕互动需要人力盯屏 | AI 大模型自动识别弹幕意图并文字/语音回复 |
| 视频素材管理混乱 | AI 自动拆分长视频 → 识别SKU+脚本 → 素材库管理 |
| 直播内容编排复杂 | 可视化编排：权重/顺序/脚本改写/特写/贴图，不修改原始素材 |

### 1.3 产品形态

```
商品知识库（多平台SKU）       视频素材库（长视频AI解析）
        │                        │
        └────────┬───────────────┘
                 ↓
         直播视频库（编排+装潢）
                 ↓
         直播间（排班+推流）
                 ↓
         直播中控台（实时监控+AI互动）
                 ↓
         数据大盘（运营总览 / 商品素材 / 商品直播）
```

### 1.4 与"数字人直播"的区别

本产品**不是**数字人实时渲染平台。核心内容载体是**商家真人录制的商品视频**。

- **不需要 GPU 集群**：不涉及实时 3D 渲染、口型同步、情感 TTS
- **不需要形象克隆**：画面是真人录播视频
- **不需要实时合成推流**：推的是预生成视频文件
- **核心壁垒在软件工程**：视频AI解析、素材编排、多账号计费、平台 API 对接

## 3. 模块架构

### 3.1 商户端模块

```
数据大盘
  📡 直播数据看板          选择已结束直播，查看单场完整运营数据
  🎞️ 商品素材数据          商品与素材交叉统计（覆盖率/时长占比）
  📡 商品直播数据          商品与直播交叉统计（播放/弹幕/弹品）

内容管理
  🛒 商品知识库           多平台SKU知识库 · AI单品/整店导入 · 富文本详情抽屉
  🎞️ 视频素材库           AI解析长视频 · 切片预览/微调 · 发布到库
  🎬 直播视频库            编排+装潢（临时调整，不修改原始素材）

直播运营
  🏠 直播间管理          房间创建 · 挂载视频 · 排班 · 开播/停播 · 配额
  📡 直播中控台          默认首页 · 竖屏画面 · 实时弹幕 · 运营操作（弹品/切品/人工回复/TTS）
  💬 互动设置           智能回复（文字/语音）· 话术模板 · 定时弹幕 · 主动互动

平台设置
  👥 账号管理           主/子账号 · 时长分配 · 使用明细
  🔗 平台接入           API凭证管理（平台级，所有直播间共用）
  ⚙️ 系统设置           全局偏好
```

> **注意**：直播排班功能已合并到「直播间管理」中，每个房间卡片内直接设置定时开关播。

### 3.2 模块关系与数据流

```
商品知识库 ── 关联SKU ────────────────┐
  (多平台知识库)                        │
视频素材库 ──AI解析──→ 已发布切片 ──→ 直播视频库（编排+装潢）
  (上传长视频)       (SKU+脚本+视频)     ├── 权重/重复次数
  20min-2h          每个切片一个单品     ├── 顺序/随机播放
                                        ├── 手动编辑脚本
                                        ├── AI改写脚本风格
                                        ├── 切片后停顿
                                        ├── 特写镜头
                                        ├── 入场转场
                                        └── 贴图
                                            │
                                            ↓ 保存为直播视频
                                       直播间（排班+推流）
                                            │
                                            ↓
                                       直播中控台（运行时）
```

## 4. 功能需求 — 商户端

选择已结束的直播场次（默认最近一场），查看单场直播的完整运营数据。

| DS-02 | 核心指标 | 8 个卡片：直播时长、累计观看（+峰值）、弹幕总数（+AI回复率）、新增粉丝、商品弹窗次数（+点击）、成交订单（+金额）、点赞数、礼物收入 | T1 |
| DS-03 | 商品讲解表现 | 该场次每个商品的讲解时长、弹窗次数、弹幕提及、点击率 | T1 |
| DS-04 | 弹幕互动趋势 | 按小时分段的柱状图 + 峰值事件标签（如"08:35 连衣裙提问"） | T1 |

**商品直播数据**

| ID | 功能 | 描述 | 优先级 |
|:---|------|------|:------:|
| DS-07 | 商品直播关联表 | 每个商品在直播间的出场与互动数据：出现直播间数、累计播放次数、累计播放时长、弹幕提及次数、弹品次数。未入排班的商品提示加入排班 | T1 |

| LC-02 | 实时弹幕面板 | 左侧固定栏，实时展示观众弹幕 + AI 自动回复，支持手动输入模拟弹幕和自动生成开关 | T1 |
| LC-03 | 实时数据监控 | 右侧面板顶部：观看人数、弹幕数、弹窗次数、订单金额，四项核心指标动态刷新（2秒间隔），+ 峰值/已播时长/AI回复率 | T1 |
| LC-04 | 手动弹品 | 运营人员手动弹出商品购买卡片，浮动在竖屏画面底部，5秒自动消失 | T1 |
| LC-05 | 切品功能 | 切换到下一个商品讲解 | T1 |
| LC-06 | 人工介入回复 | 运营人员手动输入回复内容，红色边框标识区别于 AI 回复 | T1 |
| LC-07 | TTS 语音开关 | 一键切换 TTS 语音驱动 / 原音频驱动 | T1 |
| LC-08 | 自动回复开关 | 控制 AI 是否自动回复弹幕 | T1 |
| LC-09 | BGM 音量调节 | 滑动条调节背景音乐音量 | T1 |
| LC-10 | 推流质量监控 | 画质（1080p@30fps）、码率（动态波动）、延迟、丢帧率实时显示 | T1 |

| ID | 功能 | 描述 | 优先级 |
|:---|------|------|:------:|
| RM-01 | 多直播间管理 | 自定义创建多个直播间，受套餐配额约束 | T1 |
| RM-02 | 配额展示 | 显示已用/最大直播间数 + 平台分布 tag、并发推流路数 + 平台分布 tag | T1 |
| RM-03 | RTMP 推流地址 | 每个直播间配置独立的 RTMP 推流地址 | T1 |
| RM-04 | 平台归属 | 每个直播间关联一个已接入的平台 | T1 |
| RM-05 | 挂载直播视频 | 下拉选择已编排好的直播视频（从直播视频库），显示商品数+总时长 | T1 |

### 4.5 商品知识库

商品知识库是系统核心数据资产。每个商品有一个**标准名称和 SKU**（系统内部统一标识），同时维护该商品在**不同直播平台上的独立知识库**（各平台的商品名、SKU_ID、详情描述可能不同）。这些平台商品详情是 AI 直播互动回复观众提问的知识来源。

| ID | 功能 | 描述 | 优先级 |
|:---|------|------|:------:|
| PD-01 | 商品列表 | 管理所有商品，以标准名称/SKU 为主键。表格展示每个商品在各平台的知识库接入状态（🟢已导入 / ⚪未导入，以 tag 形式显示） | T1 |
| PD-02 | 手动添加 | 手动添加标准商品（名称/SKU/分类/图片），后续可通过导入为各平台补充知识库 | T1 |
| PD-03 | 单品导入 | 选择目标平台，粘贴单个商品链接，AI 自动解析该平台商品详情页（名称/价格/规格/详情HTML/卖点），存入该商品在该平台的知识库。支持关联已有商品或创建新商品 | T1 |
| PD-04 | 整店导入 | 选择目标平台，粘贴店铺链接，AI 自动扫描店铺全部商品，列出清单供商户勾选后批量导入知识库 | T1 |
| PD-05 | 知识库详情 | 右侧抽屉展示某商品在某平台的完整知识库：平台商品名、平台 SKU_ID、平台售价、同步时间、富文本商品详情（HTML 渲染，含图片/规格/描述/买家须知）。支持刷新和删除知识库来源 | T1 |
| PD-06 | 快速导入 | 在商品列表中点击灰色平台 tag，直接触发该商品在该平台的知识库导入 | T1 |

**设计说明**：
- 商品的「标准名称/SKU」是系统内唯一标识，由商户定义
- 同一商品在不同平台可能有不同的商品名（如淘宝叫"夏季碎花裙"，抖音叫"法式连衣裙"），各自独立存储
- 富文本详情作为 AI 知识库，用于直播互动时智能回复观众关于商品的提问（尺码、面料、价格等）
- 支持平台：淘宝、抖音、快手、京东、拼多多

### 4.6 视频素材库

| ID | 功能 | 描述 | 优先级 |
|:---|------|------|:------:|
| ML-01 | 上传长视频 | 上传 20min-2h 的含多SKU商品讲解视频 | T1 |
| ML-02 | AI 智能解析 | 自动识别视频中的商品边界，切分为独立片段 | T1 |
| ML-03 | SKU 识别 | 每个切片自动识别对应的商品 SKU | T1 |
| ML-04 | 脚本提取 | 从视频音频中提取讲解脚本文字 | T1 |
| ML-05 | 切片预览 | 预览每个切片的视频片段，显示高置信度缩略图帧 | T1 |
| ML-06 | 时间微调 | 切分不准时，支持前后 ±1s/±5s 调整时间边界 | T1 |
| ML-07 | SKU 微调 | 手动修正 AI 识别错误的 SKU | T1 |
| ML-08 | 脚本微调 | 手动编辑 AI 提取的脚本内容 | T1 |
| ML-09 | 发布到库 | 确认无误后将切片发布（供直播视频库使用） | T1 |
| ML-10 | 素材库列表 | 历史解析记录列表，点击进入该视频的切片详情 | T1 |

**说明：** AI 解析进度按视频时长计算（已分析 X / 总长 Y），不预设 SKU 数量。只有解析完成的切片（SKU+脚本+视频全部完成）才显示卡片。

### 4.7 直播视频库（编排+装潢）

| ID | 功能 | 描述 | 优先级 |
|:---|------|------|:------:|
| LV-01 | 新建直播视频 | 从视频素材库加载已发布切片，创建编排 | T1 |
| LV-02 | 切片排序 | 拖拽调整商品讲解顺序 | T1 |
| LV-03 | 权重设置 | 每个切片设置权重（⭐1-5），决定重复次数 | T1 |
| LV-04 | 删除切片 | 从编排中移除某个商品 | T1 |
| LV-05 | 手动编辑脚本 | 对单个切片的讲解脚本进行手动修改 | T1 |
| LV-06 | AI 脚本改写 | 对单个切片按指定风格（亲切种草/激昂带货/专业讲解）改写脚本 | T1 |
| LV-07 | 切片后停顿 | 商品视频播放完毕后，画面定格展示商品购买卡片 X 秒，给观众下单时间，然后自动播放下一个商品。可选 0/3/5 秒。行业背景：头部直播间商品之间通常有 2-5 秒留白，用于弹品卡片展示和购买引导，而非秒切下一个视频 | T1 |

| LV-09 | 入场转场 | 商品切换时的过渡动画，替代生硬的直接切换。可选：无转场（直接切换）/ 品牌Logo动画（3秒）/ 关注引导（5秒，如"还没关注的宝宝点个关注~"）| T1 |
| LV-14 | 停顿期内容 | 配置停顿期间的画面内容：定格最后一帧 / 定格+自动弹出商品购买卡片 / 定格+自定义引导画面（如"点击小黄车立即购买"）+ 可选倒计时显示 | T1 |
| LV-10 | 贴图 | 每个切片可配置贴图（商品信息卡片/限时优惠倒计时） | T1 |
| LV-11 | 播放模式 | 全局设置：顺序播放 / 随机播放 | T1 |
| LV-12 | 竖屏预览 | 选中切片后，中间展示手机竖屏渲染预览 | T1 |
| LV-13 | 保存直播视频 | 保存编排结果，供直播间排班使用 | T1 |

**核心设计原则：** 所有在直播视频库中的调整都是**临时调整**，不修改视频素材库中已发布的原始切片。直播视频是一个独立的编排实体。

## 5. 功能需求 — 系统管理端

系统管理端是平台的"上帝视角"，分为三大板块：**核心配置**（控制商户端功能可用性）、**监控运维**（跨商户实时监控与告警处置）、**管理**（商户/账号/系统设置）。

## 8. 核心业务流程

### 8.1 素材准备流程

```
上传长视频 (20min-2h, 含多SKU)
        │
        ▼
  AI 智能解析（按视频时长推进）
        │
        ├──→ 识别商品边界，切分为独立片段
        ├──→ 识别每个片段对应的 SKU
        └──→ 提取讲解脚本文字
                │
                ▼
        切片预览 + 微调（±秒调整 / 修正SKU / 编辑脚本）
                │
                ▼
        确认 → 发布到视频素材库
```

### 8.2 直播视频编排流程

```
新建直播视频 → 从素材库加载已发布切片
        │
        ├──→ 调整顺序、权重（重复次数）
        ├──→ 每个切片独立调整：
        │      ├── 手动编辑脚本 / AI改写
        │      ├── 停顿、特写、转场、贴图
        │      └── 竖屏预览实时反馈
        │
        ▼
    保存直播视频（所有调整为临时，不修改原始素材）
```

### 8.3 直播间搭建与推流流程

```
淘宝后台创建直播间 → 获取 RTMP 推流地址
        │
        ▼
本系统：创建直播间 → 填入 RTMP 地址 → 关联直播视频
        │
        ▼
排班设置 → 定时开关播
        │
        ▼
一键推流 → 直播中控台实时监控
```

### 8.4 直播运行流程

```
播单循环播放编排好的商品视频
        │
        ├──→ 到达某商品讲解片段 → 自动弹品（淘宝 API）
        │
        └──→ 观众发送弹幕 → AI 大模型分析意图
                │
                ├──→ 文字回复（平台聊天区）
                └──→ 语音回复（TTS → 混入推流音频）
```

### 8.5 账号管理流程

```
主账号充值时长
        │
        ├──→ 创建子账号
        │        ├──→ 共享主账号时长（默认）
        │        └──→ 分配独立时长
        ├──→ 到期提醒 → 续费
        └──→ 查看使用明细 → 导出下载
```

## 9. 非功能性需求

### 9.1 性能

| 指标 | 要求 |
|------|------|
| 推流延迟 | < 5 秒（观众侧感知延迟） |
| 弹幕回复延迟 | 文字 < 3 秒，语音 < 8 秒 |
| 视频切换 | 无缝衔接，无明显黑屏或卡顿 |
| 并发直播间 | ≥ 50 路/服务器 |

### 9.2 兼容性

| 维度 | 要求 |
|------|------|
| 浏览器 | Chrome 90+, Edge 90+ |
| 视频格式 | MP4 (H.264), 1080p |
| 音频格式 | AAC, 48kHz |
| 推流协议 | RTMP |

### 9.3 安全与合规

| 要求 | 说明 |
|------|------|
| 平台合规 | 符合各平台 PC 端电脑开播规范 |
| 数据传输 | HTTPS 全链路加密 |
| 账号安全 | 主账号/子账号权限隔离 |
