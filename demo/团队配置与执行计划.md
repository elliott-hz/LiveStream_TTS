# 团队配置与执行计划

> **规模目标**：5000 视频/周 · 2000 路同时推流
> **团队上限**：10 人（不含技术总监）
> **技术栈**：Python / Go · gRPC · Kafka · PostgreSQL · K8s
> **前端**：Demo 阶段用 HTML 原型直出，后端兼顾简单页面；正式版另议

---

## 一、团队配置（10 人 · 全部研发）

```
技术总监（不占 HC）
  │
  ├── 资深研发 ×2       系统架构、核心技术决策、代码 Review
  │                     一人偏后端+基础设施，一人偏 AI+媒体管线
  │
  ├── 高级研发 ×3       自有人力，核心模块开发
  │                     业务服务 / 视频解析 / 推流+互动 / 前端页面
  │
  └── 中级研发 ×5       可外包补充，Feature 开发、API 对接、页面
                        （人员不足时优先找靠谱外包团队）
```

> **暂不设专职 DevOps**：资深研发兼任基础设施（K8s/CI/CD/监控），初期够用。
> **全部全栈**：不做前后端分离，每人能独立完成从前端页面到后端 API 的完整功能。

## 二、执行 Roadmap（4 个月）

### Month 1：团队搭建 + 架构落地

| 事项 | 说明 |
|------|------|
| **团队组建** | 全栈负责人 + DevOps 先到岗；W4 前自有核心人员到齐（≥ 50% = 4 人），其余岗位可临时外包 |
| **架构设计** | 整体技术架构确定 → 子系统架构设计完成 → gRPC Proto 定义完成 → DB Schema 定义完成 |
| **基础设施** | K8s 集群搭建、CI/CD 流水线、开发环境、代码仓库 |
| **技术验证** | gateway↔platform 调通、LLM API 联调、FFmpeg RTMP 推流验证 |

**里程碑**：团队基本到齐、infra 就绪、核心链路跑通

### Month 2：所有子系统同时研发（基础模块+核心模块）→ PoC 核心闭环

| 事项 | 说明 |
|------|------|
| **商品知识库 + 视频素材库** | AI 商品导入、多平台知识库、视频上传+解析触发 |
| **视频解析 Pipeline** | Option A 全 API：ASR→场景检测→切割→脚本，异步返回 |
| **直播间管理 + 直播视频库** | 创建/挂载/排班/开播/停播、视频编排编辑器 |
| **直播中控台 + 互动设置** | 竖屏三栏、实时弹幕、运营操作、智能回复配置 |

**里程碑**：上传→解析→编排→推流→中控台监控，全链路闭环可演示

### Month 3：全部功能完成并联调 → 交付（具备接外部第一个客户条件）

| 事项 | 说明 |
|------|------|
| **平台系统管理** | 平台接入配置、功能开关、运营监控大屏、告警处置、商户管理、系统设置 |
| **数据大盘** | 直播数据看板、商品素材数据、商品直播数据 |
| **全链路联调** | 端到端集成测试、Bug 修复、API 对齐 |
| **PoC Demo 交付** | Staging 环境部署，完整功能可对外演示 |

**里程碑**：两套完整系统 + 全部微服务 + 外部 API 联调通过

### Month 4：性能优化 & 质量优化 → 新客全部走新链路

| 事项 | 说明 |
|------|------|
| **性能压测与优化** | 2000 路推流压测、弹幕 QPS 压测、GPU 调度优化 |
| **GPU 迁移** | 视频解析 Option A→B，云 GPU 竞价实例部署 Whisper+CLIP |
| **质量优化** | 场景检测准确率提升、AI 回复质量调优、断流重连稳定性 |
| **新客切新链路** | 新接入商户全部走新系统，老客逐步迁移 |

**里程碑**：新客全部走新链路，系统稳定运行

## 三、技术风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|:---:|------|
| 场景检测准确率不达标 | 商户需大量手动微调 | W6 即开始验证，不达标 W14 迁 GPU 自研 CLIP 微调 |
| LLM API 成本超预期 | 月费远超 ¥260K | W5 开始做话术模板匹配，减少 LLM 调用；评估本地小模型 |
| 2000 路推流带宽不足 | 推流卡顿/断流 | W13 压测验证，提前预留云带宽弹性扩容 |
| GPU 竞价实例被抢占 | 视频解析任务中断 | 任务支持断点续跑；混用竞价+包月实例 |
| 团队磨合/离职 | 进度延误 | 全栈负责人 W1 到岗；核心模块文档化；Pair Programming |
